# 卡牌桌面 - 指令技术规格 v4.2

## 1. 核心设计理念：指令与场景

为了实现由角色驱动的高度动态和可扩展的游戏体验，理解指令在不同游戏场景下的应用至关重要。

-   **[地图节点信息](./ai_guides/map_node_info.md)**: **(核心必读)** 此文档解释了不同地图节点（敌人、事件、商店等）的含义，并提供了在这些场景下应如何使用 `[Event:Modify]` 指令来给予奖励或触发事件的**具体范例**。

-   **[游戏指令体系](./ai_guides/game_commands.md)**:
    -   **职责**: 管理和驱动**牌局内部**的流程。
    -   **用途**: 用于开始/结束一场牌局、洗牌、发牌、下注，以及通过 `ModifyCard` 功能直接修改卡牌属性。

-   **[事件指令体系](./ai_guides/event_commands.md)**:
    -   **职责**: 直接、即时地修改**游戏世界**的状态数据。
    -   **用途**: 这是实现所有奖励、惩罚、道具效果和状态变化的核心工具。

-   **[道具指令体系](./ai_guides/item_commands.md)**:
    -   **职责**: 管理道具的生命周期，包括创造和使用。

## 2. 通用指令结构

所有指令都由一个方括号 `[]` 包裹。方括号内包含两个部分，用逗号 `,` 分隔。

### 2.1 核心结构

`[<指令头>, <数据块>]`

#### 第一部分: 指令头 (必须)
-   格式: `类别:类型`
-   示例: `Game:Start` 或 `Event:Modify`
-   `类别` 必须是 `Game`, `Action`, `Event` 或 `Item` 之一。
-   `类型` 是具体的指令名称。

#### 第二部分: 数据块 (通常是必须的)
-   格式: `data:<JSON对象>`
-   `data:` 是一个固定的关键字，后面紧跟一个冒号 `:`。
-   `<JSON对象>` 是一个严格格式的JSON对象，它包含了执行该指令所需的所有参数。
    -   **重要**: 整个JSON对象，包括花括号 `{}`，是 `data:` 关键字的值。
    -   **JSON语法**: 所有键 (keys) 和字符串值 (string values) 都必须用双引号 `"` 包裹。

### 2.2 完整指令示例

这是一个完整的、格式正确的 `Game:Start` 指令：

[Game:Start, data:{"game_type": "TexasHoldem", "players": ["{{user}}", "拦路劫匪"], "initial_state": { "name": "拦路劫匪", "play_style": "Aggressive", "chips": 800, "hand": [] }}]
